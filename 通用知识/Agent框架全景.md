# AI Agent 开源框架全景（2025）

## 一、Agent 编排框架

### 1. LangChain
- **定位**：最主流的 LLM 应用开发框架，模块化组件 + 预构建 Agent 架构
- **核心概念**：
  - **Chain**：多步骤工作流的顺序调用链
  - **Agent**：LLM 作为推理引擎，动态决定下一步行动
  - **Tool**：访问外部 API、数据库、自定义函数
  - **Memory**：多轮对话状态保持（短期/长期记忆）
- **适用场景**：简单聊天机器人、线性 RAG 管道、快速原型开发
- **生态优势**：集成 OpenAI/Anthropic/Google 等多家 LLM，连接器丰富

### 2. LangGraph
- **定位**：LangChain 生态的 Agent 编排框架，图结构驱动
- **核心概念**：
  - **Node（节点）**：代表任务或动作
  - **Edge（边）**：定义任务间流转，支持条件分支
  - **State（状态）**：内置持久化状态管理
- **关键特性**：
  - 支持循环、条件逻辑、非线性工作流
  - 检查点（Checkpoint）+ 回溯 + 人工干预（Human-in-the-loop）
  - 多 Agent 协作：Agent 作为图节点，消息传递 + 任务委托
- **vs LangChain**：LangChain 适合线性流程，LangGraph 适合复杂有状态多 Agent 系统
- **面试高频**：两者如何互补？何时选 LangGraph？

### 3. CrewAI
- **定位**：角色扮演多智能体框架，强调团队协作
- **特色**：
  - 每个 Agent 分配角色（Role）、背景故事（Backstory）、目标（Goal）
  - 模拟真实团队协作流程
  - 建立在 LangChain 之上
- **适用场景**：需要明确工作流控制和角色分工的多 Agent 系统
- **面试点**：如何设计角色分配？工作流编排策略？

### 4. AutoGen（微软）
- **定位**：多 Agent 对话框架，异步任务执行
- **特色**：
  - 对话驱动的 Agent 交互
  - 支持 Human-in-the-loop 监督
  - 强大的代码执行和调试能力
  - 低代码工作室选项
- **适用场景**：快速原型、研究、复杂迭代问题求解
- **面试点**：Agent 路由、多 Agent 协作的涌现能力

### 5. Dify
- **定位**：LLM 应用开发平台，无代码/低代码
- **特色**：
  - 可视化工作流构建器
  - 内置 RAG 管道和知识库
  - 内置长期记忆和上下文管理
  - 集成多家 LLM 供应商
- **适用场景**：快速构建 AI 应用、混合技术背景团队
- **局限**：复杂计算任务支持较弱

### 6. OpenAI Agents SDK
- **定位**：OpenAI 官方轻量级 Python Agent 框架（2025.3 发布）
- **特色**：原生支持 OpenAI 模型，简洁 API

### 7. Google Agent Development Kit (ADK)
- **定位**：Google 官方 Agent 开发工具包

### 8. Langflow
- **定位**：低代码可视化框架，简化 Agent 和 RAG 工作流开发

## 二、RAG 框架

### 1. LlamaIndex
- **定位**：LLM 数据框架，专注数据摄取、索引、查询
- **特色**：
  - 支持多模态内容（文本、表格、图片）
  - 丰富的数据连接器
  - 多种索引类型（向量、关键词、知识图谱）
- **适用场景**：复杂数据结构的 RAG 应用

### 2. Haystack（Deepset）
- **定位**：企业级 RAG 框架
- **特色**：
  - 模块化管道架构
  - 强大的评估工具
  - 生产级部署支持

### 3. RAGFlow
- **定位**：深度文档理解 RAG 引擎
- **特色**：擅长从复杂文档（PDF、表格、扫描件）提取结构化信息

### 4. DSPy（斯坦福 NLP）
- **定位**：用"编程"代替"提示"的框架
- **特色**：
  - 声明式 Python 代码创建模块化检索系统
  - 自动优化提示和权重
  - 自改进的检索管道

### 5. Pathway
- **定位**：高吞吐量低延迟 RAG 框架
- **特色**：350+ 数据源连接器，适合大规模部署

## 三、向量数据库（RAG 必备）

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| **Pinecone** | 全托管，简单易用 | 快速上手，中小规模 |
| **Weaviate** | 开源，支持混合搜索 | 需要自定义部署 |
| **Chroma** | 轻量级，嵌入式 | 本地开发、原型验证 |
| **Milvus** | 高性能，支持十亿级向量 | 大规模生产环境 |
| **Qdrant** | Rust 编写，高性能 | 性能敏感场景 |
| **FAISS**（Meta） | 库级别，非数据库 | 研究、嵌入现有系统 |
| **pgvector** | PostgreSQL 扩展 | 已有 PG 基础设施 |

## 四、框架选型决策树

```
需求分析
├── 简单 RAG / 聊天机器人 → LangChain + LlamaIndex
├── 复杂多步骤工作流 → LangGraph
├── 多 Agent 角色协作 → CrewAI
├── 研究 / 快速原型 → AutoGen
├── 非技术团队快速搭建 → Dify / Langflow
├── 企业级 RAG → Haystack / Pathway
└── 自动优化检索 → DSPy
```
