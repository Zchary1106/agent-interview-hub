## 三、2025-2026 国内考察新增考点

### 🔥 必须掌握的新考点（2025年后高频）

**1. MCP / Skill / Function Call 三者区别（阿里/字节/蚂蚁必考）**

| 概念 | 范围 | 标准化程度 | 适用场景 |
|---|---|---|---|
| Function Calling | 单个模型厂商 | 各家不统一 | 简单工具调用 |
| Skill | 框架内定义 | 框架内标准 | LangChain/AutoGen 内部 |
| MCP | 跨厂商、跨框架 | 开放标准 | 构建可互操作工具生态 |

**2. Agent 评测体系（字节/腾讯/百度高频）**

面试官最常见追问："你的 Agent 上线了，怎么知道它好不好？"

```
评测维度（三大方向）：
├── 能力维度
│   ├── 任务完成率（Task Success Rate）
│   ├── 步骤效率比（Steps Used / Min Steps Required）
│   └── 工具调用准确率（Tool Call Precision）
├── 安全维度
│   ├── 幻觉发生率（Hallucination Rate）
│   ├── 有害输出率（Safety Violation Rate）
│   └── Prompt 注入抵抗率
└── 工程维度
    ├── 平均响应延迟（P50/P99）
    ├── Token 消耗量（成本）
    └── 系统稳定性（超时率、错误率）

评测方法：
- 离线沙箱：标准任务集，自动跑评测，快但不够真实
- 在线 AB 实验：真实流量，慢但反映用户体验
- 红队测试：专门构造边界 case，测安全性
```

**3. Vibe Coding（阿里、字节 2025年起必考）**

"Vibe Coding"是指用 AI 辅助工具（Cursor / Copilot / Claude Code）进行编程的工作方式。**面试可能要求现场演示**：

备考建议：
- 实际用 Cursor / Claude Code 完成过完整项目
- 能说清楚：什么时候信任 AI 输出，什么时候需要审查
- 了解 Prompt → 代码 → 测试 → 迭代 的 AI 辅助循环

**4. 幻觉-行动放大问题（阿里/字节/腾讯高频）**

> 面试官版本："Agent 的幻觉和单次 LLM 的幻觉有什么本质区别？"

核心答案：Agent 的幻觉会被**行动执行放大**。单次 LLM 幻觉只影响文本输出，Agent 幻觉会触发工具调用（删文件、发邮件、转账），造成不可逆损失。

防御方案：
1. 行动前确认（高危操作 Human-in-the-loop）
2. 沙箱执行（先在虚拟环境验证）
3. 输出结构化约束（Structured Output + Schema 验证）
4. 最小权限原则（Agent 只拥有当前任务必需的工具权限）

## 五、共同高频面试题 TOP 10

以下是 2026 年国内大厂 AI Agent 岗面试最高频的 10 道题，**无论投哪家都要准备**：

1. **ReAct 框架详解**：三个核心步骤（Thought/Action/Observation）及终止条件是什么？
2. **Function Calling vs MCP**：二者解决什么问题？各有什么局限？
3. **Agent 记忆系统分层设计**：工作记忆/会话记忆/长期记忆各存什么、如何协作？
4. **RAG 全流程**：从文档到答案，每个环节怎么优化？
5. **Agent 幻觉-行动放大**：为什么比普通 LLM 幻觉更危险？如何防御？
6. **多 Agent 协作模式**：Supervisor-Worker 和 P2P 各适合什么场景？
7. **上下文窗口溢出**：Context 快满了你怎么处理？（分层记忆压缩策略）
8. **Agent 评测体系设计**：能力/安全/工程三维度，离线沙箱 vs 在线 AB？
9. **系统设计**：从零设计一个支持 5000 并发的企业级客服 Agent 系统
10. **Vibe Coding 实操**：现场用 AI 工具实现一个指定 Agent 功能

## 六、30天国内大厂冲刺计划

### 第1周：基础打牢
- Java/Python 语言基础（针对目标公司）
- Redis + MySQL + 分布式系统（即使 AI 岗也考）
- LeetCode 每日一题（Medium，贵在坚持）

### 第2周：AI Agent 核心
- ReAct / Plan-and-Execute / Multi-Agent 三个模式精通
- RAG 全链路：自己动手实现一个（不用框架）
- Function Calling + MCP 深入理解，实现一个简单 MCP Server

### 第3周：工程化 + 系统设计
- 企业级 Agent 系统设计模板（并发/成本/安全三维度）
- Agent 评测体系设计（能在白板上完整画出来）
- 幻觉防御方案、输出格式鲁棒性

### 第4周：公司定向 + 模拟面试
- 针对目标公司做定向准备（见上方各公司清单）
- 找同学互相模拟面试，练习回答的结构化程度
- 准备 2-3 个能深挖的 Agent 项目故事
