## ⚠️ 字节面试关键洞察

1. **一面就很硬核**：16 道题覆盖全链路，从多模态到 MCP 到项目细节
2. **必须能演示项目**：不是说说就行，要现场展示实现细节
3. **A2A vs MCP 是新热点**：字节已经在面试中考这个了
4. **AI 辅助开发经验**：会问"AI 贡献了多少代码"——说明他们重视 AI Coding 实践
5. **手撕代码偏 AI**：不只是 LeetCode，还要手写位置编码、多头注意力
6. **DeepSeek R1 是必知**：要了解其推理能力和强化学习方法

## 📌 面经 4：字节 Agent 开发岗四面复盘（2026年5月）
**来源**: 稀土掘金 juejin.cn 2026-05-xx | 结果：Offer

**21 道核心考点（按面试顺序）：**

**一面 - 大模型基础 + Agent 设计（60min）**
1. Transformer 中 Self-Attention 的时间复杂度是多少？能否优化？
2. Decoder-only 架构为何成为主流 LLM 选择？
3. RoPE 位置编码原理？相比 ALiBi 有什么优势？
4. ReAct 框架的 Thought/Action/Observation 循环，终止条件是什么？
5. Agent 的记忆系统如何分层设计（工作记忆/会话记忆/长期记忆）？
6. **System Design**：如何构建一个内容营销 Agent，输入品牌关键词，自动生成多平台投放方案

**二面 - 多 Agent 协同 + 评测体系（55min）**
7. Multi-Agent 系统的三种通信模式：A2A、Supervisor-Worker、Blackboard？
8. 记忆系统的长期记忆如何提取？查询效率怎么保证？记忆冲突怎么更新？
9. 构建 Agent 评测体系的方法：离线沙箱 vs 在线 AB 实验各有什么优缺点？
10. 评测数据集构建策略：真实数据/蒸馏数据/人工构造难点各适用什么场景？
11. **手撕**：实现 Multi-head Attention（Python，不用框架）

**三面 - 工程落地 + 框架深度（50min）**
12. LangChain vs 自研框架：你在什么情况下会选择放弃 LangChain 自研？
13. Prompt Engineering 中 CoT / Few-shot / System Prompt 的优先级和使用策略？
14. RAG 系统：如何评估检索质量（Recall@K / MRR / NDCG）？
15. 如何处理 RAG 中的"知识冲突"（检索到的内容与模型训练知识冲突）？
16. **系统设计**：设计字节内部的"研发 AI 助手"，覆盖从需求到上线的全流程

**四面（总监面）- 战略视野 + 职业规划**
17. 你认为 2026-2027 年 Agent 技术的最大突破点在哪？
18. 字节的产品矩阵中哪个你觉得最适合做 Agent 化改造？
19. 如果给你一个 Agent 项目，你会怎么组织团队和技术分工？
20. 你理解的"工业级 Agent"和"Demo 级 Agent"的核心差距是什么？
21. 反问时间

**面试者总结：**
> 字节的面试风格：深挖原理 + 工程实践 + 前沿视野 三管齐下。纯背八股没用，面试官会一层一层追问到你不会为止。核心建议：**至少要有一个完整的从 0 到上线的 Agent 项目，能讲清楚每一个技术选型背后的 trade-off。**

## ⚠️ 字节面试关键洞察（2025-2026 更新版）

- **约 30% 后端岗已要求大模型能力**，即使非纯 AI 岗也会问 Agent 基础
- **工程落地是核心区分**："能讲清楚 LangChain 缺点并说出替代方案" >> "熟悉 LangChain"
- **评测体系是加分神器**：面试官普遍反馈很少有候选人能讲清楚 Agent 的评测方案
- **Coze 平台经验加分**：字节自研低代码 Agent 平台，了解其架构有加分
