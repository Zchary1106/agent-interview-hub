## 📌 面经 6：阿里 AI Agent 二面（2026年4月，社招）
**来源**: AtomGit 开源社区复盘 | 结果：Offer（P7）

**面试题目：**
1. **Agent 幻觉-行动放大问题**：为什么 Agent 的幻觉比单次 LLM 调用危害更大？如何通过架构设计降低风险？
2. **ReAct vs Plan-and-Execute**：分别适用什么场景？你在项目中怎么选择的？
3. **MCP 和 Function Calling 的区别**：MCP 解决了 Function Calling 的什么痛点？
4. **Skill 和 Function Call 和 MCP 的关系**：三者是什么关系？面试官追问了很久
5. **System Design**：设计一个企业级客服 Agent，要求支持 10万+ 知识库文档、多轮对话、工具调用、并发 5000
6. **手撕**：用 Python 实现一个简单的 ReAct Agent Loop（不用框架）

**关键参考答案（第3题）：**
> Function Calling 是各家厂商各自定义的工具调用接口，无法互通。MCP 是开放标准协议（Anthropic 发起），定义了 Host/Client/Server 三层架构，用 JSON-RPC 通信，任何 MCP Server 可以被任何支持 MCP 的客户端调用，解决了工具生态碎片化问题。类比 USB 接口统一了外设生态。
