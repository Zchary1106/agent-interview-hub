## 一、Agent 核心面试题

### Q: 什么是 Agentic AI？与传统 AI 的区别？
**参考答案：**
- **传统 AI**：被动响应，输入→输出，无自主决策
- **Agentic AI**：具备自主性的 AI 系统
  - 自主设定子目标
  - 主动规划执行路径
  - 使用工具与环境交互
  - 从反馈中学习和调整
- 核心区别：**自主性（Autonomy）** 和 **持续交互（Continuous Interaction）**

### Q: Agent 的"规划、记忆与反思"机制？
**参考答案：**
- **规划（Planning）**：
  - 任务分解（Task Decomposition）
  - 子目标生成与排序
  - 方法：CoT、ToT、GoT、Plan-and-Execute
- **记忆（Memory）**：
  - 短期：当前对话上下文（In-context）
  - 长期：向量数据库持久化（Embedding + Retrieval）
  - 工作记忆：当前任务中间状态
- **反思（Reflection）**：
  - 执行后自我评估结果质量
  - 分析失败原因并调整策略
  - 框架：Reflexion、Self-Refine

### Q: Agent 记忆管理有哪些难点？
**参考答案：**
1. **容量限制**：上下文窗口有限，需要选择性保留
2. **检索准确性**：长期记忆检索可能引入噪声
3. **一致性维护**：记忆更新可能产生矛盾
4. **遗忘机制**：何时丢弃过时信息
5. **隐私安全**：敏感信息的存储与访问控制
6. **跨会话持续性**：如何在多轮会话中保持上下文连贯

### Q: Agentic Workflow 如何设计？
**参考答案：**
核心组件：
1. **任务解析器**：理解用户意图，转化为结构化目标
2. **规划器**：将目标分解为可执行步骤的 DAG
3. **调度器**：决定执行顺序（串行/并行/条件分支）
4. **执行器**：调用工具/LLM 完成每个步骤
5. **观察器**：监控执行结果，判断是否达标
6. **反馈环**：失败时触发重规划或人工介入

设计原则：
- 可观测：每步都有日志和状态
- 可恢复：支持断点续执行
- 可配置：workflow 通过配置而非硬编码定义

## 二、大模型与深度学习

### Q: 主流开源大模型体系及特点？
**参考答案：**
| 模型 | 特点 | 适用场景 |
|------|------|---------|
| LLaMA 3 | Meta 开源，GQA，多语言 | 通用生成 |
| Qwen 2.5 | 阿里，中文强，长上下文 | 中文场景 |
| DeepSeek V3 | MoE 架构，性价比高 | 推理/编码 |
| Mistral/Mixtral | 欧洲开源，MoE | 轻量部署 |
| GLM-4 | 智谱，中英双语 | 国内企业应用 |
| Yi | 零一万物，长上下文 | 长文档处理 |

### Q: 向量数据库了解吗？DiskANN 索引细节？
**参考答案：**
向量数据库：存储和检索高维向量的专用数据库（Milvus、Pinecone、Qdrant、FAISS）

**DiskANN**（微软）：
- 基于 Vamana 图索引算法
- 核心思想：将索引存储在 SSD 上而非全部内存
- 构建：随机初始化图 → 贪心搜索最近邻 → 修剪边 → 保持度数约束
- 查询：内存中存储压缩向量做粗筛，SSD 读取原始向量做精排
- 优势：10 亿级向量规模，单机 64GB 内存即可

### Q: 深度学习算法：GAN、GNN、强化学习？
**参考答案：**
- **GAN**：生成器 vs 判别器对抗训练，生成逼真样本
- **GNN**：在图结构数据上做消息传递和聚合（GCN、GAT、GraphSAGE）
- **强化学习**：
  - DQN：Q-learning + 深度网络，离散动作空间
  - PPO：策略梯度 + clip 约束，连续/离散都行，训练稳定
  - SAC：最大熵 RL，鼓励探索，连续动作空间

## 三、系统与工程化

### Q: 如何搭建高并发 AI 服务？
**参考答案：**
1. **模型服务**：vLLM / TGI / Triton Inference Server
2. **负载均衡**：Nginx / K8s Ingress，按 GPU 利用率路由
3. **异步处理**：请求队列 + 消息中间件（Kafka/RabbitMQ）
4. **缓存**：语义缓存（相似 query 命中缓存）+ KV Cache 复用
5. **弹性伸缩**：K8s HPA 根据 QPS/GPU 利用率自动扩缩容
6. **流式输出**：SSE/WebSocket 降低首 token 延迟感知

### Q: AI 与数据库结合的看法？
**参考答案：**
- **Text-to-SQL**：自然语言查询数据库，Agent 生成并执行 SQL
- **向量索引集成**：PostgreSQL pgvector、MySQL 向量扩展
- **AI 优化数据库**：智能索引推荐、查询优化、异常检测
- **数据库服务 AI**：作为 Agent 的工具层，提供结构化数据访问
