# 快手 AI Agent - 面试题 & 面经

## 面试特点
- 非常注重 RAG 项目细节
- Agent 记忆机制是必考
- 手撕代码相对简单
- 面试官会先介绍组内业务

---

## 真实面试题（牛客实录）

### RAG 深度考察
1. 做 RAG 项目时怎么评测效果？评测维度和具体指标？
2. 数据集包含什么内容，数据来源、数据格式？
3. 如何优化 RAG 的相关度和回答效果？体系化方案？
4. RAG 性能怎么提升（工程 + 算法）？
5. 上下文过长、冗余问题怎么优化？
6. 引入父子索引和 BM25 的原因？比例怎么设？
7. 做了 rerank 吗？rerank 后返回几个块？如何验证效果？

### Agent 核心
8. 项目长期记忆怎么实现？框架内部实现细节？
9. 短期记忆是什么？
10. 上下文压缩方法？每次都压缩还是存起来？
11. Agent 长记忆和短记忆怎么协同？衔接逻辑？
12. Agent 任务规划怎么做？
13. 有什么思路能让 Agent 更智能？
14. Skill 怎么实现？插件和工具的区别？
15. ReAct 模式流程？串行慢怎么解决（不考虑 PE/Sub Agent）？
16. Function Calling 怎么设计？
17. Prompt 注入攻击如何防御？
18. 工具调用的安全控制？

### 后端基础
19. 分布式限流：令牌桶 vs 漏桶 vs 滑动窗口
20. Redis 实现滑动窗口
21. LRU 原理和实现
22. 布隆过滤器原理和应用场景
23. MySQL 索引失效场景，like 查询是否失效
24. MySQL 事务隔离级别、MVCC 实现

### 手撕代码
- 全排列
- 非 Hot100 Easy 级别

---

## 面试洞察
- 快手 Agent 面试 = **RAG 深度考察 + Agent 记忆机制 + 后端基础**
- 必须能讲清楚 RAG 的评测指标和优化方案
- 上下文压缩、长短记忆协同是高频难点
- Prompt 注入防御和工具安全是新考点
