# 项目 05：AI 调试挑战

## 难度：⭐⭐⭐ | 限时：1-2 小时

## 题目描述

给你一个**故意埋了多个 bug 的代码仓库**，用 AI 工具定位并修复所有问题。

## 常见 Bug 类型

### 基础级
- 语法错误 / 拼写错误
- 变量未定义 / 类型错误
- 导入缺失
- 配置文件错误（端口、路径）

### 中级
- 竞态条件（Race condition）
- 内存泄漏
- SQL 注入漏洞
- 循环依赖
- 异步错误处理缺失

### 高级
- 级联 bug（修一个触发另一个）
- 隐藏的性能瓶颈
- 安全漏洞（XSS、CSRF）
- 数据库锁
- 编码问题（UTF-8 陷阱）

## 面试官观察重点

| 行为 | ✅ 加分 | ❌ 扣分 |
|------|--------|--------|
| 定位方式 | 看日志→读代码→缩小范围 | 瞎猜→改一行试一下 |
| AI 使用 | 给 AI 完整错误信息和上下文 | 只说"帮我修 bug" |
| 验证 | 修完跑测试确认 | 改了就说"好了" |
| 关联检查 | 修完检查同类问题 | 只修报错那一个 |
| 复盘 | 说清根因和修复逻辑 | "反正能跑了" |

## 评分标准

| 维度 | 权重 |
|------|------|
| Bug 发现率 | 30% |
| 根因分析准确性 | 25% |
| AI 协作效率 | 20% |
| 修复质量（不引入新 bug） | 15% |
| 复盘与总结 | 10% |

## 准备建议

1. 练习阅读 stack trace 和错误日志
2. 熟悉常见的 Web 安全漏洞
3. 学会给 AI 提供精确的调试上下文
4. 修完一个 bug 后习惯性检查同类问题
