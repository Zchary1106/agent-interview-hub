# 蚂蚁集团 AI Agent - 面试题 & 面经

## 面试特点
- 对 Agent 工程化能力要求极高
- 会问非常细的实现细节
- 强调生产环境的稳定性和容错

---

## 真实面试题

### Agent 系统设计
1. Agent 系统 Prompt 如何设计和迭代？做过 Prompt 自动优化吗？
2. 用户提出不完整请求时，如何补全用户意图？
3. 构建 Agent 时遇到过哪些瓶颈？
4. Agent 哪些模块最容易在真实业务中出问题？如何监控和定位？
5. Agent 使用了多少个外部工具？调用链条上如何保障故障容错和超时机制？
6. 工具调用失败后的 feedback 策略如何设计？
7. 多轮对话上下文状态管理如何做？高并发下如何保证一致性？

### LLM 与微调
8. LangChain 的 memory 默认机制在多用户并发中如何隔离？线程安全？
9. 微调 Llama2 时如何选择训练样本？清洗逻辑？
10. DPO 相比 SFT 的优劣？在 Agent 任务上效果提升明显吗？如何构造偏好对？
11. 服务部署在 vLLM 上的原因？KV-cache 如何加速推理？做过哪些优化？
12. Streaming 输出但延迟超标，如何折中设计？

### MCP 与工具
13. 写一个 MCP Server 的流程
14. Skill vs Function Call vs MCP 对比，Skill 有哪些特性？
15. MCP 和 Skill 哪个上下文占用大？如何写好一个工具？

### 数据与训练
16. 训练数据来自用户行为日志，如何抽取训练对话？归一化或事件抽象？
17. 模型如何"理解时间"？带时间窗口的对话系统？

### 后端基础
18. Java 线程生命周期，线程池参数选择
19. G1/CMS 垃圾回收器参数配置
20. CPU 使用率过高如何排查
21. TCP 滑动窗口协议原理
22. MySQL 索引数据结构，联合索引，索引失效场景

---

## 面试洞察
- 蚂蚁是目前 Agent 面试**问得最细**的公司之一
- Prompt 工程、工具容错、并发管理是三大核心考点
- 会问你 MCP Server 怎么写——要有实操经验
- DPO/vLLM/KV-cache 等偏底层的优化也会考
