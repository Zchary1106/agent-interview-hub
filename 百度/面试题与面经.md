## 一、大模型基础（必考重点）

### Q: Transformer 核心组件详解？
**参考答案：**
- **Self-Attention**：计算序列内每对 token 的关联度
- **Multi-Head Attention**：多个注意力头捕获不同维度的关系
- **位置编码**：弥补 Transformer 无序列感知的缺陷
  - 正弦/余弦编码、可学习编码、ROPE、ALiBi
- **FFN**：两层全连接 + 激活函数，提供非线性变换
- **Layer Norm + Residual Connection**：稳定训练

### Q: Encoder-Only / Decoder-Only / Encoder-Decoder 各适合什么任务？
**参考答案：**
- **Encoder-Only**（BERT）：理解任务（分类、NER、QA）
- **Decoder-Only**（GPT）：生成任务（文本生成、对话）—— 当前主流
- **Encoder-Decoder**（T5）：序列到序列（翻译、摘要）

### Q: RLHF 的完整流程？DPO vs PPO？
**参考答案：**
RLHF 三阶段：
1. SFT：在高质量数据上微调
2. Reward Model 训练：人类标注偏好对，训练奖励模型
3. PPO 优化：用 RM 的信号通过 PPO 算法优化策略模型

DPO vs PPO：
- PPO：需要额外的 Reward Model + Critic Model，训练复杂
- DPO：直接从偏好对学习，无需 RM，更简单高效
- DPO 通过重参数化将 RM 的目标直接转化为策略优化目标

### Q: 大模型幻觉问题？如何缓解？
**参考答案：**
原因：
- 训练数据有噪声/偏差
- 模型过度自信的生成
- 缺乏实时/专业知识

工程缓解方法：
1. RAG 引入外部知识
2. 提示工程（要求引用来源、思维链）
3. 自我一致性检查（多次采样投票）
4. 输出后校验（事实核查 Agent）
5. 微调高质量数据
6. 温度调低减少随机性

### Q: 模型复读问题的原因和解决方法？
**参考答案：**
原因：注意力集中在近期 token、训练数据中的重复模式
解决：
- Repetition Penalty（重复惩罚）
- Frequency/Presence Penalty
- 训练时数据去重
- 后处理检测截断

### Q: 混合专家模型（MoE）如何扩大参数但不增推理成本？
**参考答案：**
- 多个 Expert FFN 模块 + Router 网络
- 每次推理只激活 top-k 个 Expert（如 2/8）
- 总参数量大但激活参数少
- 挑战：负载均衡、Expert 坍缩、通信开销

## 二、Agent 设计

### Q: 让你设计一个 Agent，会怎么做？
**参考答案：**
1. **定义问题**：明确 Agent 的目标和边界
2. **选择 LLM**：根据任务复杂度选模型
3. **设计工具集**：确定 Agent 需要的外部能力
4. **记忆方案**：短期（对话历史）+ 长期（向量 DB）
5. **编排策略**：ReAct / Plan-and-Execute / 多 Agent
6. **评估体系**：准确率、任务完成率、延迟、成本
7. **兜底与安全**：超时处理、错误恢复、内容安全

### Q: 如何让 Agent 支持多语言交互？
**参考答案：**
- 选择多语言 LLM（如 GPT-4、Qwen）
- 工具描述和 System Prompt 用英文（模型理解最好）
- 用户输入/输出保持原始语言
- 特定语言的 RAG 知识库
- 语言检测 → 路由到对应处理链路

## 三、训练与优化

### Q: 分布式训练策略？
**参考答案：**
- **数据并行（DP）**：每个 GPU 完整模型副本，数据分片
- **模型并行（MP）**：模型按层/张量切分到不同 GPU
- **流水线并行（PP）**：模型按阶段切分，micro-batch 流水
- **ZeRO**：分布式优化器状态/梯度/参数，三个阶段逐步减少冗余
- 实践：通常混合使用（3D 并行）
